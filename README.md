# Tarea Semana 3: Programación Tradicional vs POO

Estudiante: Anderson Johao Rodriguez Espinosa

## Descripción
Este proyecto cumple con la actividad de la Semana 3 de la asignatura Programación Orientada a Objetos, comparando dos enfoques para gestionar información de mascotas.

- **`programacion_tradicional/`**: Solución implementada mediante funciones y variables, sin utilizar clases ni objetos.
- **`programacion_poo/`**: Solución implementada mediante la clase `Mascota`, utilizando atributos y métodos para representar objetos de forma estructurada.

## Reflexión
La Programación Tradicional nos permite resolver problemas de forma rápida mediante funciones lineales. Sin embargo, al escalar el proyecto, la Programación Orientada a Objetos (POO) demuestra ser superior al permitir la abstracción y el encapsulamiento y esto nos facilita la creación de estructuras complejas, el mantenimiento del código y la reutilización de objetos, lo cual es fundamental en el desarrollo de software
